XML とは？

  • Extensible Markup Language の略
  • 文章マークアップ言語かつデータ交換フォーマット
  • SGML のサブセット

XML の構文

			
  <?xml version="1.0" encoding="UTF-8" ?>
  <foods>
    <food>
      <name>バナナ</name>
      <color>黄色</color>
    </food>
	
    <food>
      <name>リンゴ</name>
      <color>赤</color>
    </food>
  </foods>

SGML とは？

  • Standard Generalized Markup Language の略
  • 言語を作る言語
  • テキストに付加情報を記述できるマークアップ言語

XML の歴史

  • 1996年7月

    W3C がチームを発足

  • 1996年11月

    W3C ドラフトの発表

  • 1998年2月

    XML1.0 の勧告

  • 2004年2月

    XML1.1 の勧告

XML の設計目標

  1. インターネット上で利用可能であること
  2. 様々なアプリケーションをサポートすること
  3. SGML と互換性があること
  4. XML 文書を処理するプログラムが簡単に書けること
  5. オプションは必要最低限であること
  6. 文書は人が読みやすく、合理的であること
  7. 設計する準備が簡単にできること
  8. 設計が簡潔であること
  9. 文書が簡単に作成できること
  10. XML マークアップの使いやすさはあまり重要でないこと

XML の現状

  • XML2.0 は議論は行われたが未勧告

JSON とは？

  • JavaScript Object Notation の略
  • データ交換フォーマット
  • JavaScript オブジェクト表記構文のサブセット

JSON の構文

			
  {
      "メンバー表":[
          {
              "氏名":"田中太郎",
              "年齢":18
          },
          {
              "氏名":"山田花子"
              "年齢":19
          }
      ]   
  }

XML と JSON の違い

  • 文書マークアップ言語であるかの違い
  • ネームスペースがあるかないかの違い
  • オブジェクトモデルに直接マップできるかの違い

JSON の歴史

  • 2002年

    json.org で仕様が公開

  • 2006年

    RFC4627 が公開

  • 2013年

    ECMA-404 の発表

  • 2014年

    RFC7159 が公開、同時に RFC4627 の廃止

  • 2017年

    RFC8259 と ECMA-404 の発表

まとめ

  • XML は SGML のサブセットであり JSON は JavaScript のサブセット
  • XML の文書マークアップ言語としての側面
  • JSON はオブジェクトモデルに直接マップ可能
